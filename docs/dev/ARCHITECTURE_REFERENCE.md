# LlamaRunner Pro - Project Structure

## Architecture Reference

This project has been refactored according to separation of concerns principles.

### Directory Structure

- src/backend/core/ - Main business logic
- src/backend/services/ - Services and business logic
- src/backend/api/ - REST API endpoints
- src/backend/models/ - Data models
- src/backend/proxy/ - AI proxy management
- src/backend/monitoring/ - Monitoring and metrics
- src/backend/patterns/ - Design patterns
- src/frontend/ - User interface
- src/shared/ - Shared code
- config/ - Configuration
- scripts/ - Launch scripts
- tests/ - Tests organized by type
- docs/ - Structured documentation
- tools/ - Deployment tools

## Benefits of this Architecture

1. Separation of Concerns - Each module has a clear responsibility
2. Maintainability - Code easier to maintain and test
3. Scalability - Easier to add new features
4. Readability - Clear and intuitive structure
5. Testability - More targeted and effective tests
